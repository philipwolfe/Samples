System Requirements
  • The samples require a machine running IIS 4.0 or higher.
  • The samples demonstrating database access require SQL Server 7.0 (or higher) or MSDE.  If SQL Server is not installed on your machine, you will be given the option to install MSDE. 
  • The samples assume that a local SQL Server is using the credentials: username=sa and password= <blank>. The samples will not run properly if your database server uses different credentials.

Configuring the SQL Server Databases
Four databases are used throughout the samples: The two default databases: pubs and northwind; And two custom databases: GrocerToGo and Portal. Install these databases using SQL Server Enterprise Manager. The samples setup program copies the database files into the SQL Server's default data directory.
    For example:
    C:\Program Files\Microsoft SQL Server\MSSQL\Data
     

Modifying Config.web
The config.web file in the quickstart directory is modified to reflect the samples installation directory.
. . .
<sourceview>
    <add key="root" value="C:\Program Files\Microsoft.Net\FrameworkSDK\Samples\QuickStart" />
</sourceview>
. . .
 
Adding the Registry Keys
The addition of four registry keys are required.
    DWORD HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\COMPlus\Switches\ABooleanSwitch = 1
    DWORD HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\COMPlus\Switches\ATraceSwitch = 4
    DWORD HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\COMPlus\Switches\TraceMethods = 1
    DWORD HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\COMPlus\Switches\Arithmetic = 4
     
Compiling the Samples
To compile the samples run the NMAKE program from the samples installation directory.
    For Example:
    C:\Program Files\Microsoft.NET\FrameworkSDK\Samples> NMAKE all

    Note:  On Windows 9x, it may be necessary to run corvars.bat located in the SDK bin directory prior to running NMAKE.
