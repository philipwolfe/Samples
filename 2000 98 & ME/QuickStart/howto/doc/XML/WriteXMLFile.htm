How Do I...Write XML to a file?

This sample illustrates how to write XML to a file using the XmlTextWriter class. The writer provides a fast, forward only way of generating XML and helps you to build XML documents that conform to the W3C Extensible Markup Language (XML) 1.0 specification and the Namespaces in XML specification. This writer provides stream write to XML rather than using an object model such as the XML DOM, and so gives better performance. See How Do I...Load and use the XmlDocument (W3C DOM)?.

Typically the XmlTextWriter is used if you need to write XML as 'raw' data without the overhead of a DOM and therefore provides a faster mechanism for writing XML. The XmlWriter class is the API which provides the XML writing to file, stream or a TextWriter. This provides numerous validation and checking rules to ensure that the written XML is well formed. When certain violations occur exceptions are thrown, which should be caught. The XmlTextWriter has different constructors to specify the location to write the XML data to. In this sample we are going to write XML to the newbooks.xml file. The sample code below constructs an XmlTextWriter with a string representing the file location. 


XmlTextWriter writer = new XmlTextWriter ("newbooks.xml", null);


Dim writer As XmlTextWriter = New XmlTextWriter ("newbooks.xml", null)
C# VB  

The constructor takes the filename that you want to write to, and the encoding that you would like to generate. If encoding is 'null' it writes out UTF-8. The XML 1.0 specification details how XML documents are encoded.

Here we are going to write another fragment of the bookstore XML file. For each XML type there are corresponding XML write methods. For example, to write an element use the WriteElementString method and to write an attribute use the WriteAttribute method. For nested levels the WriteStartElement/WriteEndElement pair is used and equally for more complex attribute creation you can use the WriteStartAttribute/WriteEndAttribute pair. The code sample below creates an XML file with a single book. 


writer.Formatting = Formatting.Indented;
writer.WriteStartDocument(false);
writer.WriteDocType("bookstore", null, "books.dtd", null);
writer.WriteComment("This file represents another fragment of a book store inventory database");
writer.WriteStartElement("bookstore");
    writer.WriteStartElement("book", null);
        writer.WriteAttribute("genre","autobiography");
        writer.WriteAttribute("publicationdate","1979");
        writer.WriteAttribute("ISBN","0-7356-0562-9");
        writer.WriteElementString("title", null, "The Autobiography of Mark Twain");
        writer.WriteStartElement("Author", null);
            writer.WriteElementString("first-name", "Mark");
            writer.WriteElementString("last-name", "Twain");
        writer.WriteEndElement();
        writer.WriteElementString("price", "7.99");
    writer.WriteEndElement();
writer.WriteEndElement();

//Write the XML to file and close the writer
writer.Flush();
writer.Close();


writer.Formatting = System.Xml.Formatting.Indented
writer.WriteStartDocument(false)
writer.WriteDocType("bookstore", nothing, "books.dtd", nothing)
writer.WriteComment("This file represents another fragment of a book store inventory database")
writer.WriteStartElement("bookstore")
    writer.WriteStartElement("book", nothing)
        writer.WriteAttribute("genre","autobiography")
        writer.WriteAttribute("publicationdate","1979")
        writer.WriteAttribute("ISBN","0-7356-0562-9")
        writer.WriteElementString("title", nothing, "The Autobiography of Mark Twain")
        writer.WriteStartElement("Author", nothing)
            writer.WriteElementString("first-name", "Mark")
            writer.WriteElementString("last-name", "Twain")
        writer.WriteEndElement()
        writer.WriteElementString("price", "7.99")
    writer.WriteEndElement()
writer.WriteEndElement()

'Write the XML to file and close the writer
writer.Flush()
writer.Close()
C# VB  

First the XML declaration with the version "1.0" is written with the WriteStartDocument method. This method, although optional, must be the first write call before any others writes, otherwise an exception is thrown. Next the document type is written with the name "bookstore" with the method WriteDocType. The 3rd parameter writes SYSTEM "books.dtd" to indicate that there is an external DTD to validate against. The Formatting property sets the output to be formatted. This causes child elements to be indented using the Indentation and IndentChar properties. The Flush method persists the XML to file (here only the Close method is really required, however there are occassions whereby the XML generated so far needs to be persisted and the writer reused)

To check the output from the XmlTextWriter, we can perform a round trip test by reading in the generated file with an XmlTextReader to validate that the XML is well formed.

Summary

  1. The XmlTextWriter provides a fast, forward only way of generating XML.
  2. The XmlTextWriter helps you to write XML documents that conform to the W3C Extensible Markup Language (XML) 1.0 specification and the Namespaces in XML specification.
  3. The XmlTextWriter provides constructors to write XML to a file, a stream or a TextWriter.
  4. For each XML type there are corresponding XML write methods.
