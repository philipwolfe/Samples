ADO.NET: Executing a Command

Commands are issued against databases to do actions against data stores. For example, you may execute a command that inserts data, or a command that deletes data. For more information on easily moving data in and out of databases read the topic "Updating data in a database". Commands include any command that can be issued against a database, and in the case of the ADOCommand, can be data store specific. For example, you may issue a stored procedure call for a command, or perhaps a command to "set quoted_identifier on". Whatever the command may be, the ADOCommand or SQLCommand can be used to get the command to your back end data store.

ADO classic allowed several different ways to issue commands, through the Command object, Connection object or Recordset object. In ADO.NET, it is the Command object only that executes commands.

In order to issue a command against a database, the command object must have two basic things: a Connection and CommandText. Both of these can be set in the constructor. In order to be able to execute the command, the Connection has to be open and not in fetching state: 


String insertCommand;
insertCommand = "Insert into Region (RegionID, RegionDescription) VALUES (100, 'Description')";
SQLCommand myCommand = new SQLCommand(insertCommand, myConnection);


Dim insertCommand As String
insertCommand = "Insert into Region (RegionID, RegionDescription) VALUES (100, 'Description')"
Dim myCommand As SQLCommand = New SQLCommand(insertCommand, myConnection)
C# VB  

To execute the command you simply call the ExecuteNonQuery method. 


myCommand.ExecuteNonQuery();


myCommand.ExecuteNonQuery()
C# VB  

The ADOCommand and SQLCommand objects have a parameter collections which behaves just like the parameter collections from ADO classic. You can pass your parameters in-line: 


myCommand.CommandText = "MyStoredProc 'CustId'";


myCommand.CommandText = "MyStoredProc 'CustId'"
C# VB  

Or by using the parameters collection: 


workParam = myCommand.Parameters.Add(new SQLParameter("@CustomerID", SQLDataType.NChar, 5));
workParam.Value = "NewID";


workParam = myCommand.Parameters.Add(New SQLParameter("@CustomerID", SQLDataType.NChar, 5))
workParam.Value = "NewID"
C# VB  

For more information about using parameters and out parameters see Getting Out Params from a Stored Procedure
