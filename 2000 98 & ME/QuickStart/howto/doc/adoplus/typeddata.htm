ADO.NET: Working with Typed Data

ADO classic code provides late-bound access to values within its recordset through weakly typed variables. ADO.NET code provides the ability to access the data held in the DataSet through a "strongly typed" metaphor. This means you can access tables and columns that are part of the DataSet with user-friendly names and strongly typed variables. Therefore, rather then writing the ADO classic code: 


AdoRecordset.Find("Customerid = 'ALFKI'");
AdoRecordSet.Fields["FirstName"].Value  = "Bob";


AdoRecordset.Find("Customerid = 'ALFKI'")
AdoRecordSet.Fields("FirstName").Value  = "Bob"
C# VB  

with a strongly typed DataSet you write the ADO.NET code:


CustomerDataSet.Customers["ALFKI"].CustomerName = "Bob";


CustomerDataSet.Customers("ALFKI").CustomerName = "Bob"
C# VB  

Additionally, the strongly typed DataSet provides access to row values as the correct, strongly typed, value. In using ADO, variant types are used when assigning and retrieving data. If the value that you assigned was of the wrong type, ADO would give you a run-time error. With ADO.NET, if the value is of type integer, and you attempt to assign a string, you will get a compile-time error.

Given an XML schema that complies with the XSD standard, you can generate a strongly typed DataSet using the XSD.exe tool provided with this SDK. The syntax for generating a DataSet using this tool is:

xsd.exe /d /l:C# {XSDSchemaFileName.xsd}

The /d directive directs the tool to generate a DataSet; the /l: directive specifies the language to use -- either "C#" or "VB."

Once the data is loaded using the FillDataSet method, the code loops through each customer in the "customers" table using the typed customerRow object. Note the direct access to the "customerid" field, as opposed to accessing it through the Fields collection.
