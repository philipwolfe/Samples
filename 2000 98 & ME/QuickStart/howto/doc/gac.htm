How Do I...install an assembly into the global assembly cache?

The global assembly cache is a machine wide store used to hold assemblies that are meant to be shared by several applications on the machine.  When the Common Language Runtime releases its final version, assemblies will be installed in the cache using a new version of the Windows Installer.  For this pre-release, a utility included with this SDK named SN can be used for this purpose.  There is also a Windows shell extension that allows you to look at the contents of the assembly cache.  To see what's in your cache, just navigate to <%windir%>\assembly.  

All assemblies in the cache must have a shared name.  The makefile included with this sample builds an assembly with a shared name and installs it into the assembly cache.
