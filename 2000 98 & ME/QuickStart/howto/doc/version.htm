How Do I...change which version of an assembly I bind to?

This sample demonstrates how to use an application configuration file to bind to a specific version of a dependent assembly.  By default, the version of a dependent assembly that will be loaded at runtime is the version specified in your assembly manifest (subject to automatic QFE policy).  If you'd like to run with a different version, you specify  "Binding Policy" in your application configuration file.  For an executable, the configuration file resides in the same directory as the exe and has the same name (minus ".exe") with a ".cfg" extension.  For this sample the executable is called math.exe and the configuration file is called math.cfg.  The relevent poriton of the cfg file looks like this: 

<BindingPolicy>
<BindingRedir Name="Calcr"
                 Originator="32ab4ba45e0a69a1"
                 Version="5.0.0.0" VersionNew="6.0.0.0"
                 UseLatestBuildRevision="yes"/>
</BindingPolicy>
The "BindingRedir" attribute of the "BindingPolicy" tag specifies that all references to version 5 of Calcr within this application should use version 6 instead. Note that version policy applies only to assemblies with shared names.
