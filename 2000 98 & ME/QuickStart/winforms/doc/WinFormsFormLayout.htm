Dynamic Layout in Windows Forms

The controls on a Form can be dynamically resized as the Form resizes.

Windows Forms provides  three ways to control the layout of your Form:

  • Anchoring
  • Docking
  • Custom
Anchoring

When a control is anchored to an edge of its container, the distance between the control and the specified edge remains constant when the container resizes. If a control is anchored to the right edge of its container, the distance between the right edge of the control and the right edge of the container remains constant when the container resizes. A control can be anchored to any combination of the edges of a control. If the control is anchored to opposite edges of its container (for example, to the top and bottom), it resizes when the container resizes.

For example, if you anchor a TextBox control to the left and right sides of a Form, then as the Form is resized the width of the TextBox will change.

The following code anchors a TextBox to the top, left and right sides of it container:


textBox1.Anchor = AnchorStyles.TopLeftRight;


textBox1.Anchor = AnchorStyles.TopLeftRight

Docking

When a control is docked to the edge of its container, it remains in contact with that edge when the container resizes. For example, Windows Explorer docks its TreeView control to the left side of the window and its ListView control to the right side of the window. If multiple controls are docked to one edge, one is docked to the edge of the container, and the others are docked as close as possible to the edge without covering each other.

The following code docks a Panel to the Left of a its container:


Panel1.Dock = DockStyle.Left;


Panel1.Dock = DockStyle.Left

Custom Layout

You can write your own Layout manager using the Layout event exposed on the Control class. This event is raised whenever any event occurs that would cause the control to layout child controls. These events include resize, show/hide child controls, add/remove child controls. This event should be used to perform any custom layout logic.
