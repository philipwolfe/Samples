.NET Framework: Send & Receive Data via HTTP

This solution demonstrates how to send & receive data using HTTP and streams using the System.Net classes.

Featured Highlights

There are 4 command buttons, 2 text boxes, and a picture box. Additionally, there is a virtual directory named VDir1 that receives and returns data. The first 3 buttons show how to send and receive data from a file. The last button shows how to send text back and forth to the web application. When using the fourth button, the text in the editable text box is sent to the web server, and the returned text is displayed in the read-only text box.

The Windows application relies upon 4 ASPX files living in the VDir1 directory.

Requirements

Requires the Trial or Release version of Visual Studio .NET Professional (or greater).

Running the Sample

The application communicates with a virtual directory served up by IIS. By default, the directory is located in a sub-directory below the solution. The virtual directory must be configured using the Internet Services Manager. Name the virtual directory VDir1, associate it with the VDir1 directly, and accept all defaults.

File I/O is performed in & out of the files using FileStreams. Network I/O is performed between client & server using Request and Response streams.

There are three data files used in this sammple:

  • datafile.txt
  • mypic.jpg
  • myxmlfile.xml

All three are expected to be in the VDir1 directory. In addition, a copy of the file datafile.txt should be in the same location as the main exe (which by default is the bin directory).
