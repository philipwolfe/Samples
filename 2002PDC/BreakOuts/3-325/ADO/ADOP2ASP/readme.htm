Sample: Using ADO+ functionality from an ASP page using VBScript

 

ActiveX Data objects (ADO) is the standard model to access data across different platforms and sources: Microsoft Access databases, spreadsheets and SQL tables. It is a language independent Automation server.

This sample requires IIS and SQL server have been set up and running on the machine.

 

What the sample does:

  • Connects to an SQL server using the connection object and its methods and parameters from the ASP page using VBScript. 
  • Accesses an SQL database and query an SQL table from the unmanaged asp page using the SQL DataSetCommand and SQLCommand objects for better performance with the SQL  server.
  • Accesses fields from the table.
  • Dispays data field information from the SQL table on the web page.
  • Executes queries and stored procedure from the web page using the imported metadata on the ADOPlus contained in System.Data.dll.
  • Executes stored procedures to select and update data in a table.
  • Creates an ADO table object to bind data to a table element and displays the fields on the page.

 

How to run:

  1. Set an environment variable LOCATEDLL to point to the location where System.Data.DLL is installed.
  2. Run build.bat.
  3. Internet Explorer starts and displays the initial records in the SQL table ‘titles’. Then a second table is displayed after executing the Update command and executing the ADDNEW command. The title of the book ‘Life Without Fear’ will change to ‘A Good News’ and a new record with the book title 'Come with us to the Moon' will be added to the SQL table.   

5.  Close Internet Explorer. The records will be changed back to their initial state.
