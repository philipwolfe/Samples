Sample Overview

This sample demonstrates the integration between the Web Services Platform and COM+ 1.0 Services.

COM+ Services is a Microsoft technology that allows the development of highly scalable, multi-tier distributed applications.  Successful commercial Web sites are experiencing ten-fold and even one hundred-fold increase in order volume as more and more business moves to the Internet. It is clear that scalable software systems that remain ready to serve at any time and never fails is a requirement.

This sample demonstrates that your existing technology investment is protected whilst allowing you to move your software forward to take advantage of the innovations in the Web Services Platform.

 

The Bank Sample demonstrates a simple business problem of performing database operations and preserving the integrity of the database through distributed transactions.

The Bank App Client uses the MoveMoney component to transfer money between two accounts and uses two Account components to perform the transfer. The SQL Server database holds the account rows and also keeps track of receipts. The Account components perform the actual work of updating the database table. MoveMoney and Account are marked to require a transaction. When MoveMoney is created a transaction is started by COM+ Services. When the Account components are create ,this transaction is flowed to them, so that when the transfer is completed and MoveMoney returns, the transaction is commited and the database table reflects the transfer.

The components are provides in multiple languages:

·         Bank App client is written in VB6

·         MoveMoney is provide multiple languages: VB6, VB7 and CSharp

·         Account is also provide multiple languages: VB6, VB7 and CSharp

The components written in VB7 and CSharp demonstrate that an existing component can be replaced without amending the client application at all, by performing the same tasks using the same interfaces.

Location of the Sample

This sample is contained in the ComServices subdirectory

Preparing the Environment

The environment needs to be prepared in order for the demo sample to run.

Database

You must create a SQL server database called “bank”. Use the SQL Server Enterprise Manager to create the database. Here are detailed steps:

  • Start -> Programs -> Microsoft SQL Server 7.0 -> Enterprise Manager
  • Open Microsoft SQL Servers
  • Open SQL Server Group
  • Open your Server
  • Right Click on Databases and click “New Database . . .”
  • Enter “Bank” for the database name
  • Click OK

The SQL Server Database should have been created.

File DSN

You must create a File DSN “BankSample” that points to the “bank” SQL Server Database. The File DSN “BankSample” is use by the sample components to update account balances and keep track of receipt numbers. Here are detailed steps:

  • Select Start -> Settings -> Control Panel -> Administrative Tools -> Data Sources (ODBC)
  • Click on the “File DSN” tab
  • Click the Add button
  • Select SQL Server from the list of drivers and Click Next
  • Type in the name of the Data Source “BankSample” and Click Next

·         Click Finish

  • Enter “BankSample” in the Description EditBox
  • Enter the name of the machine that has the Bank database or select from the drop down list. Click Next
  • Leave authentication to the default “With Windows NT authentication using the network login ID”. Click Next.
  • Change the default database to “Bank”. Click Next
  • Click Finish.
  • Click Test Data Source
  • Click OK
  • Click OK
  • Click OK

The File DSN “BankSample” should have been created.

Building the VB6 MoveMoney and Account Components

First build the VB6 MoveMoney and Account components. Here are detailed steps:

  • Using Windows explorer locate the Visual Basic Project VB6Bank and double click it.
  • You should now be in the Visual Basic 6.0 development environment select File -> Make vbacct.dll
  • Select Ok to make the dll.
  • You may need to select Yes to overwrite the existing vbacct.dll
  • Exit Visual Basic.

The VB6 MoveMoney and Account components should have been created in the vbacct.dll

Creating the COM+ 1.0 Application and adding the COM components (VB6) to it

The MoveMoney and Account components in the various languages need to be hosted in a COM+ 1.0 Application. The installSamplebank.vbs creates the “BankApp” COM+ 1.0 Application and adds the VB6 components to it. It also configures the VB6 components with the appropriate service requirements e.g. Requires Transactions. The installSampleBank.vbs can be found in the BankAdmin directory, which is a subdirectory of the ComServices directory.

·         Open a command prompt

·         Navigate to the ComServices directory.

·         Change to the BankAdmin directory

·         Run the command: installSamplebank.Vbs

Once the script is completed you should see a Message Box displaying “Done installing Sample Bank”. Click OK.

 

Build the VB7 and C# Components

The VB7 and C# Components need to be built and added to the COM+ 1.0 “BankApp” Application. Here are detailed instructions:

·         Open a command prompt

·         Navigate to the ComServices directory.

  • Run the command: nmake -a –fmakefile

The VB7 and CSharp components should have been build and added to the COM+ 1.0 “BankApp” Application.

Build and Start the VB6 Bank App Client

To build the legacy client:

  • Using Windows explorer locate and then double click on Samples\ComServices\vb6bankClient\vbbank.vbp
  • Use Run -> Start with full compile to execute the app

The Bank App Client should be running.

Run the Bank App Client Sample

The step above “Build and Start the VB6 Bank App Client” should have started the Bank App Client. Below are a number of scenarios to try out.

Credit Transaction – Account

In this Scenario we credit account number 1 with $1.

·         Set the Account number to 1

·         Set Amount to 1

·         Set Transaction type to “Credit”

·         Set Component to Account

·         Set Language to VB6

·         Click Submit

The result box should state that a credit has been applied to account 1 and should display the new balance.

Try the Credit operation in VB7

·         Set Language to VB7

·         Click Submit

Try the Credit operation in C#

·         Set Language to C#

·         Click Submit

Transfer Transaction – Money

In this Scenario we transfer $5 from account number 1 to account number 2.

·         Set Component to MoveMoney

·         Set Transaction type to “Transfer”

·         Set the Account number to 1

·         Set the Transfer to Account to 2

·         Set Amount to 5

·         Set Language to VB6

·         Click Submit

The result box should state that a credit has been applied to account 2 and should display the new balance for account 2. It should also show that a debit has been applied to account 1 and should display the new balance for account1

Try the Transfer operation in VB7

·         Set Language to VB7

·         Click Submit

Try the Transfer operation in C#

·         Set Language to C#

·         Click Submit
