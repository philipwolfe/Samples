Release Information for the Iguana Tech Preview

 

What is Iguana

 

Iguana is the ActiveState internal codename for PerlScript targeted at the Microsoft .NET Platform.

 

Release Purpose

 

This release provides a technical preview of the Iguana codebase to PDC participants.  It is provided for evaluation purposes only. It contains prerelease code, documentation and specifications, and is not at the level of performance and compatibility of the final product offering

 

What is included in the Tech Preview

 

·        A copy Perl 5.6.0, compiled with debugging options and some changes to the official distribution (roughly equivalent to ActivePerl build 613).

·        PerlScript 2.0 compiler

·        Sample/Test programs

 

Not included are the following ActivePerl features:

(ActiveX) PerlScript, PerlIS, PerlEz, HTML help, additional modules

 

Where to get an updated version of the Iguana Tech Preview

 

An updated version of this tech preview will be made available after the PDC on the web at:

http://www.ActiveState.com/complus

 

How to install the Iguana Tech Preview

 

Only install the IguanaTech Preview on computers that are not used for production or mission-critical data. If you install Iguana Tech Preview on a computer used for production, you do so at your own risk.

Just doubleclick on the file Iguana-5.6.0.1257.msi.  The installation requires about 20 MB of disk space. It installs into C:\Iguana by default.

The Microsoft .NET Platform SDK must be installed on your machine before you install the Iguana Tech Preview.

 

Known Issues

 

·        Don’t install the Iguana Tech Preview over an existing installation of ActivePerl.  Make sure C:\Iguana is on your PATH before any other Perl installation.

 

·        The installer adds C:\Iguana\bin to your CORPATH system environment variable.  Make sure it doesn’t get hidden by a user environment variable.

 

·        The .NET Platform runtime generates security exceptions when you install this code on a network drive.  It works correctly on local drives.

 

·        When you uninstall the Iguana Tech Preview, the samples directory will not be removed if you built the sample code in there.

 

·        The CORPATH setting might not be restored correctly during uninstallation of the IGUANA Tech Preview.

 

 

Directories

 

C:\Iguana\bin

C:\Iguana\lib

C:\Iguana\site

 

These include the standard Perl interpreter with the standard modules as well as the PerlScript 2.0 compiler and the PMRT.DLL runtime library.

 

C:\Iguana\samples

 

Various sample programs (see below).

 

PerlScript Compiler

 

The PerlScript compiler frontend is in C:\Iguana\bin\psc.bat.  Run “psc -help” to get basic documentation.

 

Sample Walkthrough

 

To compile all the samples, got to the C:\Iguana\samples directory and run “nmake”.  The samples demonstrate the following features:

fac.pl

 

Simple factorial calculation to test that recursion with local variable allocation works correctly.

environ.pl

 

Sample “CLS consumer” program.  It calls both class and instance methods.  It also introduces strongly typed variables (CLS data types).  The output of this program is very similar to the output of “set”.

property.pl

 

This script demonstrates property access.  It uses the “hash ref” syntax to both get and set named properties and uses “array ref” syntax for indexed properties.

 

pinvoke.pl

 

This is the “platform invoke” demo:  It uses the GetTempPath() and MessageBox() APIs to display the TEMP path to the user in a messagebox.  The GetTempPath() call returns values not only as the function return value but also in one of the parameters.

classtest.cs / MyClass.pl

 

Sample “CLS producer” program.  MyClass.pl implements two classes:

MyBaseClass and MyDerivedClass.

·        MyBaseClass has a constructor that requires a parameter.  It also contains a protected member variable as well as an overloaded method Foo().  Both externally visible Foo() methods will resolve to the same Perl method Foo.

 

·        MyDerivedClass derives from MyBaseClass.  The MyDerivedClass constructor (parameterless) calls the MyBaseClass constructor, passing in a numeric constant to the base constructor.  The hello() method throws an exception (PerlScript.DieException).

 

 

·        classtest.cs instantiates instances of both classes and calls the overloaded (and overridden) method Foo().  It also calls the hello() method in MyDerivedClass and catches the DieException.

except.pl

 

This sample tests exception handling:  It uses an eval block to catch an exception generated by an outside class and throws an exception object itself (as opposed to dying with a string).  It can either catch this exception itself or pass it on to the runtime.
