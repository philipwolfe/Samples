Managed Python Status

Mark Hammond (MarkH@ActiveState.com) - June 2000.

Managed Python is an implementation of the Python language (www.python.org) for the COM+ 2.0 environment.

Updates for the Python COM+ compiler and documentation can be found at http://www.ActiveState.com/complus

This document is intended as overview of the status of Managed Python. For information on the Python language, or any of the existing implementations of the language, see the Python web site.

For information on building and running the Managed Python compiler, please see the README.TXT in the compiler distribution.

Accomplishments To Date

·        The compiler supports around 95% of the core Python language to some degree.  A comprehensive test suite is in-place for these core language features.

·        Python classes can be written that derive from arbitrary COM+ Types and override almost any virtual method (CLS compliant or not).  Arbitrary COM+ interfaces can be implemented.  These features work for both the standard class library, or types and interfaces defined in 3rd party assemblies.

·        The Python compiler creates managed assemblies that can be used by any other COM+ language.  Thus, for example, C-Sharp can derive from a class implemented in Python.

·        The Python compiler can compile and run the standard Python benchmark program.  This benchmark program is not trivial, and highlights the level of conformance to the Python language.  The fact the benchmark runs at all is a major milestone.

·        The compiler itself is written in Python, currently using the existing “unmanaged” Python implementation and the COM interoperability layers to use Reflection::Emit().

·        The compiler generates fully debuggable code.  The programmer can single-step through their Python source code.

·        There is an extensive test suite for the compiler and runtime.  Experienced Python programmers who browse this Python source code will see that even many of the obscure language features are implemented.

Accomplishments yet to be done!

·        Support for the standard Python library is quite weak.  For this reason, many existing Python programs will not run.  However, new programs that take advantage of the COM+ library instead of the Python library should work quite well.

·        The compiler, and the runtime code, is quite slow.  Addressing the speed issues are a high-priority.

·        Python class and instance semantics are not fully supported, particularly where Python clashes with COM+ semantics.

·        No support for custom attributes, meaning certain functionality from some tools (e.g., VS7, WFC) is not available.

·        A few core language features are not implemented, such as unlimited size integers, complex numbers, string formatting and so forth.

Managed Python – Technical Overview

This section provides a detailed technical analysis of the Managed Python project.

Introduction to Python

Python is a highly flexible and dynamic language.  Variables are not declared, and indeed the language does not have any syntax for type declarations.  The very nature of Python means that many of its semantics are defined at runtime.  For example, the Python statement:

foo()

Simply means “look up the name ‘foo’ in the current namespace, and apply the ‘call’ operation on it”.  There is no implication that ‘foo’ is a function – just some object with ‘call’ semantics (which a function object obviously has.)

In addition to this dynamic nature, the Python language itself is quite small and simple.  Much of the language functionality comes from the standard Python Library – a set of Python modules that provide various services.  For example, the os module performs all file system operations; the time module performs all time-of-day manipulations, etc.  The Python programmer must explicitly import these modules and then use the services it provides.

Therefore, Managed Python can be viewed as consisting of the following components:

·                    IL Compiler, which uses the Reflection::Emit() object model to create managed assemblies.  A significant part of the code generated by the compiler consists of calls to the Managed Python Runtime.

·                    The Managed Python Runtime is a COM+ assembly, written in C-Sharp.  This runtime implements the Python language semantics for COM+ objects (for example, defines that Managed COM+ classes and methods are Python ‘callable’)

·                    The standard Python library.  Although only a few of these modules currently exist for Managed Python, the ones that do have been written in Managed Python itself, simply by using the IL Compiler and Managed Python Runtime.  This category includes the Python builtin functions, which are a set of functions available to all Python programs without any module imports.

Each of these components is at a different state of development, and has differing levels of support for Python and COM+.
