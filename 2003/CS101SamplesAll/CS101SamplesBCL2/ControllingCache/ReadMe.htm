Controlling Caching

The .NET Framework 2.0 introduces the new namespace "System.Net.Cache" which defines cache policies for resources obtained using the WebRequest and HttpWebRequest classes.

This sample allows the user to select which caching policy will be in effect for a URL when the "GoButton" is clicked.  A description of the selected caching policy is displayed for the selected policy.  The raw HttpResponse is displayed in a RichTextBox after the "GoButton" is clicked.

The status label will display whether the reponse came from the cache or not.  It is interesting to note how setting the cache policy to "NoCacheNoStore" for a web request clears out the cache for a specific page and then setting the cache policy to "CacheOnly" for a web request for the same url throws the correct exception that there is no response in the cache.

This sample requires the following to be installed:

  • Visual Studio 2005
