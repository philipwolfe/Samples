DataProtection API

The .NET Framework provides access to the data protection API (DPAPI), which allows you to encrypt data using information from the current user account or computer. When you use the DPAPI, you alleviate the difficult problem of explicitly generating and storing a cryptographic key.

This sample demonstrates the use of both the ProtectedMemory and ProtectedData objects.  This functionality is available in Microsoft Windows XP and later operating systems.

Use the ProtectedMemory class to encrypt an array of in-memory bytes.  You can specify that memory encrypted by the current process can be decrypted by the current process only, by all processes, or from the same user context. This sample demonstrates the current process only option.  See the MemoryProtectionScope enumeration for a detailed description of ProtectedMemory options.

Use the ProtectedData class to encrypt a copy of an array of bytes. In this sample, the encrypted data is written to a file, "Data.dat" and decrypted by reading this file.  You can specify that data encrypted by the current user account can be decrypted only by the same user account, or you can specify that data encrypted by the current user account can be decrypted by any account on the computer. This sample demonstrates the same user account option.  See the DataProtectionScope enumeration for a detailed description of ProtectedData options.

This sample requires the following to be installed:

  • Visual Studio 2005
  • Microsoft Windows XP and later operating systems
