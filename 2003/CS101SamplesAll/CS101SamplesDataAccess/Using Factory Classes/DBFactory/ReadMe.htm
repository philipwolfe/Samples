Factory Classes Sample

Factory Classes are a set of classes that allow you to generate generic code to access a disparate set of data sources without having to change and recompile code. This leads to better code reuse and realibility.

In this example, you will see how to use the new DbProviderFactory class to connect to different data sources. Generic coding is accomplished with specific information coming from the app.config file. This allows you to create code once and switch between data sources without having to recompile.

To run this example, select the Data source you want to connect to click the Get Data button. This accesses the app.config file to retrieve the connection string for the specific data source. The data from that data source is shown in the data grid and the Provider name, connection string and performance results are shown in the the Test Results group box.

This sample requires the following to be installed:
  • Visual Studio 2005.
  • SQL Server Express 2005.
