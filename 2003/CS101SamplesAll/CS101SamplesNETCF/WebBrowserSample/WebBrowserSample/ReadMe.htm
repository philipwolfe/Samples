WebBrowser Sample

The .NET Compact Framework (.NET CF) 2.0 and Visual Studio 2005 make it easier to deploy smart device applications.

This sample demonstrates the new WebBrowser control, the new Splitter control and the new LinkLabel control.

With the WebBrowser control you can view both offline and online content.  This sample demonstrates both types of content, but to access online content, an online connection must exist.  The emulator will not display online content.  You must deploy this sample application to a physical device.

Deployment is the process by which you transfer an application to the target device.  Before you can deploy your solution, you must package it into a CAB file. A CAB file is a type of executable archive file that contains the application, dependencies such as DLLs, resources, help files, and any other files you want to include.

A CAB file project, WebBrowserCAB, has been included in the solution so the WebBrowserSample application can be easily installed on a physical device.  Please build both projects and manually copy the resulting WebBrowserCAB.CAB file to the target device (not the emulator) using your ActiveSync connection.  Tap the CAB file on the physical device to install the WebBrowserSample.exe on the PocketPC. Find and run WebBrowserSample.exe.  (Look under the "Program Files" directory.)

The Splitter Control is viewed on the application as a thin line above the browser.  Simply tap and drag this line to modify the height of the browser.

This sample requires the following to be installed:
  • Visual Studio 2005 Beta 2.
  • .NET Compact Framework v2.0 (installed on physical device).
  • Online Connection to the internet on the physical device.
