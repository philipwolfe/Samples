Working with Named Ranges in Excel

Named ranges allow you to easily reference one or more cells in code.  Contained named range values can be retrieved, updated or databound.  Named ranges expose events such as selected/deselected, changed, or double-clicked.

In this sample, three named ranges are declared at design time, and one is declared at runtime.  Each maps to a single cell.  If either of two named ranges detect changes to their contained value, a Changed event will fire, and a third named range value will be set.  For this example, salary and debt are keyed into cells, and a classification such as low/average/high/extreme is set to a third cell.  The runtime-declared named range is assigned a simple string value at sheet startup.  Named ranges make it easier to programmatically manage a worksheet similar to labels and textboxes on a Windows form.

This sample requires the following to be installed:

  • Visual Studio 2005
  • Microsoft Excel 2003
