Using the BindingSource Control

This sample demonstrates the use of the BindingSource control for browsing the Employees table in the Adventure Works databse. The BindingSource control enables you to bind controls to any of the three types of .NET data sources: a database (which, when added to a project takes the form of a DataSet), an object, or a Web service.

Three tabs are presented, each demonstrating a different BindingSource control instance. The first tab presents a user interface created at design time by dragging and dropping items from the Data Sources window.

The second tab displays a similar Form, though built programmatically. A BindingSource control is created that uses a DataTable of Employee data as its DataSource. A BindingNavigator is added to provide navigation through the DataTable.

The third tab displays a form identical in functionality to that on the second tab, but its BindingSource control uses an object rather than a DataSet as its data source.

This sample requires the following to be installed:
  • Microsoft Visual Studio 2005
  • Microsoft SQL Server 2005 Express
  • AdventureWorks database installed at the default location
