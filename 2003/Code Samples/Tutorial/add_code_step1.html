Step 1 of 7: Introduction

Thus far, we've used the Web Form designer to add controls to our form and set some properties. This should be familiar to anyone who has used Visual Basic 6.0.

With ASP.NET, our code is compiled so we get much better performance, debugging, error handling and other benefits. Visual Studio .NET uses the concept of a code-behind file to separate the presentation and logic of your page into separate files. The aspx file contains your user interface code (html and controls). The Visual Basic .NET source file containing code sits "behind" your aspx page, hence the name "code-behind".

  1. Press F7 to display the Visual Studio .NET code editor. You can also right-click the Web Form in the designer and select View CodeThis is similar to toggling between "View Code" and "View Form" when building a Visual Basic 6.0 form.

  2. At the top of the file, you will see a Class declared that has the same name as your Web Form. 
    Public Class WebForm1 Inherits System.Web.UI.Page

  3. Declare 2 variables to hold the DataSet and Dataview. Class level variables are generally declared at the top of the class. Add the following code after the line with Inherits. 
    Protected dvOrders 
As New DataView
Protected WithEvents dsOrders As New dsCustOrder

  4. Throughout this step you will add code to your code-behind file. As a general rule of thumb, just paste it between the Public Class WebForm1 and End Class statements and you will be fine.
