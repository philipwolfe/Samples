Step 2 of 5: Add Data Files

  1. To get started implementing data access in this walkthrough, download a zip file which contains the XML data file CustOrder.xml and the schema file dsCustOrder.xsd which defines the structure for the DataSet.

  2. After the download completes, extract the files to a directory of your choice but make a note of the location.

  3. Go back to Visual Studio .NET 2003 and in Solution Explorer, right-click on the OrdersWebVB project. From the submenu, choose Add - Add Existing Item. Find the CustOrder.xml and dsCustOrder.xsd files and add them to the project so we have data and a definition for our DataSet. You may need to choose to "Show All Files.

In order for Datagrid editing to work, the account that the web site uses (ASPNET user) must have necessary permissions to write to the XML file. As a good security practice, the ASPNET user has very limited permissions by default. You must explicitly grant permissions for the ASPNET user account to modify the CustOrder.xml file. These are general instructions and the process may vary slightly depending upon your operating system. For further help, contact your system administrator.

  1. In Windows Explorer, find the directory of this sample project. By default the path will be C://inetPub/wwwRoot/OrdersWebVB/

  2. The CustOrder.xml file you just added will be in this directory.

  3. Right-click the CustOrder.xml file and choose Properties

  4. Click the Security tab.

  5. Click Add and find the ASPNET local user account. Add the account to the list of accounts with permissions.

  6. With the ASPNET user account still highlighted in the list, make sure to grant at least Modify permissions

  7. Close out of the File Properties dialog box and continue building the application.

(Note: If you get stuck on this step, you can return to it later. All of the functions of this sample other than updating will still work fine.)
