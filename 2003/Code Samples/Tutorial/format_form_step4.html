Validating Web Forms using the Validation Controls

Step 4 of 4: Validation

A task often dreaded in Web applications is writing validation code. Validation code enforces rules ensuring that data entered into a form meets the application requirements for data types, ranges and so on. Often a mix of client-side and server-side script, validation logic can be hard to reuse effectively.

To remedy this issue, ASP.NET includes a set of validation controls that provide a consistent mechanism for validating your Web application forms in an intuitive way.

The CompareValidator compares the data entered against a static value or a value from another field on the form. We will use one to make sure that the start date for our order search, as entered in txtStartDate, is earlier than January 1st, 2001.

  1. Drag the CompareValidator from the Toolbox onto the Web Form.

  2. The Display property is set to Static by default. This simply means that the error message will be rendered in the spot where we place the validator.
  3. The free space under the Customers dropdown is a good spot to place the control.

  4. Set the ErrorMessage property to the error message you want to display, for instance "StartDate must be less than January 1, 2001".

  5. Set the ControlToValidate property to txtStartDate. This property displays all the form controls in a combobox, so you may also select from the list.

  6. Set the Operator property to LessThanEqual.

  7. To properly perform comparisons, the CompareValidator must know what type of data is being compared. The options for the Type property are String, Integer, Double, Date & Currency. Choose Date.

  8. Set the ValueToCompare property to 1/1/2001

  9. Save the Web Form.
