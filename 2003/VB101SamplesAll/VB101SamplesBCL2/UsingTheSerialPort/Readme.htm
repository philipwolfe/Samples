Using the Serial Port With the .NET 2.0 BCL

 

Introduction:

 

One of the new features in the .NET 2.0 BCL is the ‘System.IO.Ports.SerialPort’ class.  The ‘SerialPort’ class facilitates access to the systems RS-232 serial ports, usually identified as COM1-COMn.

 

New features demonstrated:

 

Access to the serial port parameters allow developers to set the properties of the port at runtime.  Both accessor and mutator properties are available to examine and set such parameters as baud rate, number of data bits, number of stop bits, parity, RTS, and others.  Line-oriented input and output via ‘ReadLine’ and ‘WriteLine’ are demonstrated, but the new class also contains a ‘BaseStream’ property that supports stream-oriented I/O as well.

 

Example code:

 

The example code demonstrates how properties may be examined and set for the serial port.  Serial transmit and receive functions are additionally demonstrated.

 

 

Running the example:

 

The example is available as a VS.NET 2005 console applications project in both C# and VB.NET.  In operation, the serial port is examined for relevant properties and those are displayed on the main form.  Clicking the buttons beside the ‘Baud Rate’ property causes the baud rate to change for the serial port.

Entering text in the text box provided and clicking the ‘Send’ button will send the data to the serial port buffer.  The serial data received event will be invoked after transmission completes and a received message will appear in the text box.  A null modem connector may be necessary for the example to work properly.  Most serial ports are of the DB-9 variety and the proper connector will have transmit and receive pins 2 and 3 wired together.

 

This sample requires the following to be installed:

 

Visual Studio .NET 2005

Serial port available

Null modem connector
