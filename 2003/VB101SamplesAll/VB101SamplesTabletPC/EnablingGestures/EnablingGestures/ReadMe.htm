Recognizing Gestures

The Microsoft Windows XP Tablet PC Edition 2005 provides system-level recognition of various traced gestures, ranging from basic geometric shapes (square, circle, triangle, etc.) to proof-reading symbols such as curlicues and chevrons. Gesture recognition can be turned on or off, and fine-tuned using the SetGestureStatus() method of the InkOverlay class.

 

This samples enumerates all of the system gestures and populates a CheckedListBox with their names. As gestures are checked and unchecked, listening for them is turned on or off. There is some additional processing to handle the special gestures "AllGestures" and "NoGestures" commands.

The Gesture event is fired by the InkOverlay as appropriate. The associated event arguments contain an array of Gestures with their IDs and Confidence levels. N.B.: NoGesture is often passed with a higher Confidence than the hoped-for gesture, so it's important to use a loop similar to what is shown in this sample to iterate over the passed-in Gestures and confirm that the hoped-for gesture has, in fact, a higher confidence.

The array of returned Gestures is displayed in a Label.

This sample requires the following to be installed:

  • Visual Studio 2005.
  • Microsoft Tablet PC Platform SDK.
  • Microsoft Windows XP Tablet PC Edition 2005.
