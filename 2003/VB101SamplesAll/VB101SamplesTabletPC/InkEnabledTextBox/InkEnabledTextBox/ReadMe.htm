Ink-Enabled TextBox

The Microsoft Windows XP Tablet PC Edition 2005 provides the Pen Input Panel (PIP) that automatically provides for text input using either freeform handwriting, character-by-character recognition, or a "tap"-able QWERTY keyboard. This is the simplest and often the best solution for text entry in a Tablet application.

Controls that support the Text Framework Services additionally provide the ability to correct mis-recognitions on a word-by-word basis after the initial recognition.

Finally, it is a straightforward matter to use an InkOverlay in conjunction with a Timer and the handwriting recognition APIs to create a customized ink-enabled TextBox.

This sample demonstrates these three different options. First, a simple TextBox is used in which data is entered via the PIP. Second, the TFS capabilities of a RichTextBox are available: after entering text with the PIP, select a word and the PIP will reappear with alternates for that word available.

Finally, a TextBox, InkOverlay, and Timer are used to allow ink-gathering on the TextBox's surface. After the Timer's Interval expires with no new Strokes added, the ink is recognized and the text is placed in the TextBox.

This sample requires the following to be installed:

  • Visual Studio 2005.
  • Microsoft Tablet PC Platform SDK.
  • Microsoft Windows XP Tablet PC Edition 2005.
