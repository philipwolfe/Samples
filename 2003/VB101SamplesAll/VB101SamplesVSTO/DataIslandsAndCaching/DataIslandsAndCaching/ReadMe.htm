Data Islands and Caching

Microsoft Office 2003 provides a powerful data caching ability using a feature called data islands.  This allows you to store arbitrary serializable data with your document.  This can be accessed by your customization the next time it is run, or by a server in a back-end scenario.

In this sample, you will see a method of filling a dataset, in this case using the Product table in Adventure Works, then storing the data in the document behind the scenes.  When the document initializes each time, it first attempts to read the data from the database directly.  If the database cannot be reached, most likely due to an off-network scenario, the data is retrieved from the data island, if available.  A status message in the worksheet indicates the source of the data, and whether or not it was found cached (even if it was loaded live).  The dataset is cached declaratively using the attribute, Cached.  A second cached field is cached imperatively using the StartCaching method which takes the name of an instance variable.  The ability to store network-accessed information for offline use without resorting to custom persistence is a powerful features enabling a more productive, mobile workforce.

This sample requires the following to be installed:

  • Visual Studio 2005
  • Microsoft Excel 2003
  • Adventure Works sample database installed at "C:\Program Files\Microsoft SQL Server\MSSQL\MSSQL.1\Data"
