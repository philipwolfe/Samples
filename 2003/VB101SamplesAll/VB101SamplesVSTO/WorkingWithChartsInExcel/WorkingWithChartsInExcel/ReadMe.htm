Working with Charts in Excel

Microsoft Excel offers a powerful data presentation and manipulation platform for tabular data in many formats.  In addition to the traditional row/column spreadsheet metaphor, you can also create dynamic charts to better visualize data trends.  As with other Excel objects, you can programmatically interact with charts in various ways.

In this sample, you will see two styles of charts -- a chart object embedded within a worksheet, and a chart sheet host item.  A chart sheet host item is analogous to a worksheet, but contains only a chart, and no cells.  In Visual Studio Tools for Office, chart sheets have their own designer and code-behind.   A chart object, on the other hand, is treated like any other control.  Data is bound to columns A-D from the Adventure Works vSalesPersonSalesByFiscalYears view.  The data is used as the source data for the two charts.  The Document Actions pane is used to demonstrate programatically setting the chart's caption, the chart type, and filtering a dataset bound to a ListObject to affect rendered chart data.  Note that four of the chart types, with names beginning with xlStock, require specific data arrangement in order to work, so will be ignored if selected.  Also notice that both the chart object and chart sheet are both able to be manipulated identically using the customization.  Entering a value in the "Show sales consistently above" field acts as a filter only showing rows where all values from 2002-2004 are above that target.

This sample requires the following to be installed:

  • Visual Studio 2005
  • Microsoft Excel 2003
  • Adventure Works sample database installed at "C:\Program Files\Microsoft SQL Server\MSSQL.1\MSSQL\Data
