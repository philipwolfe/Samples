Working with the Inbox in Outlook

The Outlook Inbox is where many professionals spend a large amount of time each day.  This makes it a great place to create customizations that run when new mail is received, filed, or based on explicit actions.  The Outlook object model exposes a wealth of information on mail items, and makes it easy to work with and manipulate that information.

In this sample, an Inbox customization is registered when Outlook starts up.  This exposes no interface, but monitors for new messages.  When new mail items arrive, it checks their subject and body for various phrases in order to take actions such as modifying the importance, flagging for followup, or deleting the message.  Using the NewMailEx event, you can be notified of all messages and take action using the power of the .NET Framework and the Outlook managed API.

Here are the triggers and the actions taken:

Trigger Action
Subject starts with "URGENT:" Importance = HIGH
Subject starts with "OPENME:" Displays the message
Subject starts with "FYI:" Importance = LOW
Subject starts with "FOLLOWUP:" Flags the message for followup
Subject starts with "FLAGME:" Flags the message for followup and adds a red colored flag
Body contains "This is an automated message" Deletes the message

This sample requires the following to be installed:

  • Visual Studio 2005
  • Microsoft Outlook 2003
