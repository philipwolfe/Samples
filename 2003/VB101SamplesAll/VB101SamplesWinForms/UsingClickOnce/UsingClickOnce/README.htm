Using ClickOnce

ClickOnce is a new deployment technology introduced in the .NET Framework 2.0. ClickOnce supports three deployment strategies:
  • Check for updates automatically on startup
  • Check for updates automatically after startup
  • Check for upgrades through application logic
The first two strategies are configured using Visual Studio 2005, as part of ClickOnce publishing. The third strategy involves adding code to the application to check for updates via a user interface element such as an Update Now button, or automatically by some custom logic.

This sample demonstrates implementing ClickOnce updates through application logic, in this case, a user interface that allows the user to check for and commence updates.

Important Note: This sample cannot be run in Visual Studio 2005. It must be deployed using ClickOnce, and run from the installed application. ClickOnce has been configured in this sample to install the application to a local Web site, http://localhost/UsingClickOnce. To create this Web site and deploy the application , in Visual Studio 2005 select the Build | Publish UsingClickOnce menu command. This will launch the Publish Wizard. Accept the default values, and click Finish to deploy the sample application. ClickOnce will open a browser to the deployment Web page for the application. Click Install to install the application. If a security warning dialog appears, click Install. ClickOnce will launch the application once it is installed.

To generate an update for the application, exit the application and re-publish it using the same procedure. There is no need to make any changes to the application to create an update. Once an update has been published, do not reinstall from the deployment Web page. Instead, navigate to Start | All Programs | Visual Studio 2005 101 Samples | Using ClickOnce to use the application to update itself.

This sample requires the following to be installed:
  • Visual Studio 2005
  • Internet Information Services
