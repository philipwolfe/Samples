SummaryGenerates a resource file from a .mc file
 
The .mc file has the same structure as described in the Microsoft documentaion. For conversion to a Managed Code application the follow assumptions are made:
  1. SeverityNames and FacilityNames have the following format: 
     SeverityName = (label=value:name ... )
 FacilityNames = (label=value:name ... )
    The :name element is ignored.

    The following Severity codes are predefined: 

     Success = 0x0
 Informational = 0x1
 Warning = 0x2
 Error = 0x3

    The following Facility names codes are predefined: 

     System = 0xFF
 Application = 0xFFF

  2. The LanguageNames directive has the following format: 
     LanguageNames = (language=culturename ... )
 For example: LanguageNames=(English=en German=de)

    The following Languages are predefined: 

    	English   en
	German    de
	TChinese  zv-CHT    (Chinese Traditional)
	SChinese  zv-CHS    (Chinese Simplified)
	Spanish   es
	French    fr

  3. Message text contains Managed Code formatting strings not printf strings.
  4. The \ character is used as the place holder delimitor not % .

    The following place holders are recognized: 

    	\n - for newline
	\t - for tag
	\b - for a leading or trailing space
	\0 - if at the end of the last line, do not insert a newline at the end of a message

  5. A newline character will be appended to the last line unless directed not to.
  6. Message text is terminated by a line containing only a period or a blank line.
