The handling of txt files is where resgenx differs from resgen . The message text can consist of multiple lines. For example: 
 SymbolicName:MsgID = text line1
         text line2
         text line3
A message continues until a blank or comment line is found. As with resgen leading and trailing spaces are ignored. Comment lines must start with a semicolon ";" or a pound sign "#". Comment lines cannot appear in the body of a message. The following escape sequences are recognized by ResGenX :
  • \n - Insert a newline chararcter. By default a newline character is added to the end of a message.
  • \t - Insert a tab character.
  • \b - Used to reserve leading or trailing spaces.
  • \0 - Used to suppress the insertion of a newline to the last line of a message. Must be used at the end of the last line in a message.
The SymbolicName element is optional. If specified the command line option /xref (see Process ) is used to create a cross reference of the MsgID and the SymbolicName .
