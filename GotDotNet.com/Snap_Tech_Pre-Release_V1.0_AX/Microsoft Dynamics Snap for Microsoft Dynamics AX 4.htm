INTRODUCTION


This document provides important, late-breaking information about Microsoft Dynamics Snap Technical Pre-release Version 1.0 for Microsoft DynamicsTM AX 4.0 / 2007 Microsoft® Office system, which supplements the Microsoft Dynamics Snap documentation.

The snap-ins included in this release are:

·        Business Data Lookup Snap-in

·        Vacation Management Snap-in

·        Timesheet Management Snap-in

·        Custom Report Generator Snap-in

·        Customer Journal Snap-in

For the latest set of issues and information, please check the documentation, news items and message boards at Microsoft Dynamics Snap Sandbox.

 

SYSTEM REQUIREMENTS

1.    At least 5 MB of disk space must be available on the local machine installation location.

2.    The installation files, including AXWave2setup.exe, must be copied on your local machine. Do not install from the download site or from a network share directly

3.    Microsoft Office 2007 should be installed, along with all the latest updates

4.    Microsoft .NET Framework 2.0 should be installed on the system

5.    Microsoft VSTO 2005 SE Runtime  should be installed on the system

6.    Pre-release versions of all snap-ins should be uninstalled before starting installation

7.    The snap-ins have been tested on Windows Vista Enterprise, Windows XP Professional, Windows Server 2003 editions of Windows

8.    The snap-ins have been tested on Enterprise and Standard editions of 2007 Microsoft Office system.

A detailed list of pre-requisites is given in the Microsoft Dynamics Snap Installation Guide in the docs folder.

 

INSTALLATION

 

Run the AXWave2Setup.exe and follow the instructions. For detailed installation instructions, please read Microsoft Dynamics Snap Installation Guide in the docs folder.

 

DOCUMENTATION

 

An exhaustive description of each of the Snap-ins is provided in the whitepapers in the docs folder. For latest insights into the snap-ins, please visit the message boards of the Microsoft Dynamics Snap Sandbox.

 

KNOWN ISSUES

 

The Known Issues below are categorized by the snap-in.

 

Setup

 

Installing snap-ins for one user does not install them for other users on the same machine

This is a known issue and there is no workaround.

 

At a time, only one user can have snap-ins installed on the machine

This is a known issue and there is no workaround. If another user needs to use the snap-ins, the first user must uninstall.

 

Microsoft Dynamics Snap for Business Data Lookup

 

Context sensitive menu does not show the entries for Business Data Lookup Snap-in, when invoked in Microsoft Office Outlook 2007.

          This is a known issue and there is no workaround.

 

Wild-chars, ?, % do not work in Look for box.

          This is by design and is mimicking the behavior of Microsoft Dynamics AX.

 

Document of size greater than 4 MB cant be attached to Microsoft Dynamics AX.

No workaround exists for this known issue. Users will need to take care not to try to attach documents larger than 4 MB in size.

 

If a search is made with string containing special characters like ^, &, %, etc, results might be unpredictable.

          There is no workaround for this issue.

 

If an autolookup field group is added in Groups in the config file then all the field groups after that and including that do not come in select details grid.

          This issue has been retained for fixing in future releases.

 

Sometimes clicking on the Insert from AX button causes the Attach to AX task pane to be launched.

          This issue has been retained for fixing in future releases. You can toggle between the two task-panes by clicking on their respective Links at the bottom of the task pane under the Other Microsoft Dynamics AX tasks section.

 

The BDL snap-in takes a lot of time to fetch records if they are huge in number.

          The snap-in has not yet been tuned for Optimum performance. It is planned to be taken care of in the future releases.

 

Sometime Microsoft Word crashes after the BDL snap-in is installed and used for a while.

          This issue has not been found to occur frequently and there still are no concrete  steps to reproduce it. It is expected to be nailed and resolved in future releases.

 

BDL Snap-in has not been tested on Home and Students edition of Microsoft Office 2007 RTM.

          These are not part of the targeted platforms in the Snap roadmap so far.

 

Documents get attached to some entities (e.g. Customer Invoice Journal) through BDL even though Microsoft Dynamics AX 4.0 client does not allow attaching documents to those entities.

          This issue has been retained for fixing in future releases.

 

BDL does not honor Field level security

          This issue has been retained for fixing in future releases.

 

In Microsoft Word, File->Send->Mail leads to Microsoft Outlooks new mail window to hang

          This issue has been retained for fixing in future releases.

 

Undo functionality in Microsoft Word and Microsoft Excel does not behave properly after the user inserts some data using BDL.

          This issue has been retained for fixing in future releases.

 

Invalid entries found in Config file are not detected and may cause Errors

          This issue has been retained for appropriate action in future releases.

 

Microsoft Dynamics Snap for Custom Report Generator

 

It is recommended that excel workbooks be designed such that user has permissions on all of the entities that form a part of a workbook.

The CRG gives the power to create reports which get data from a wide variety of entities and honors the security privileges as specified by CRM. If a user doesnt have permission on a particular entity, data is retrieved till that entity is encountered in the excel sheet, and then the retrieval process will stop, even though this entity may be followed by entities for which the user has permission to view data.

 

Changing the structure of the Budget Creator sheet  for e.g retrieving budget data, then inserting rows in between and then clicking on update may cause erratic behavior.

The example sheet provided is not protected. So an end user can insert rows in between retrieved data and cause erratic behavior. However, the developer of the custom report can protect the structure of the workbook (excel feature) and prevent such activity.

 

If the Number of entries in a table increase beyond the number of rows in a table, then the Formulas are not copied in newly created Cells.

Excel formulas are given for a appropriate range of cells. If the data retrieved does grow beyond this range of cells, the formulas need to be reset to get correct information. Typical examples would be SUM or AVG functions.

 

Un-installation removes the example reports in the installation folders.

The example reports are provided as examples of how this snap can be used. The excel sheet along with the xml grammar file can be saved to any location on the disk and they will work fine.

 

XML query doesnt support custom entities

As of now the infrastructure does not support, fetching data from custom entities created in the CRM system.

 

Master data and other Special structures like Array Lists are not insertable through the XML grammar.

Master data like accounts etc. are not insertable through XML grammar as of now. Pick lists might use data structures like Array Lists which are currently not updatable through the XML grammar.

 

Data pertaining to the Default Company of a user is accessible through this snap.

          This is a known issue and there is no workaround.

 

 

Microsoft Dynamics Snap for Vacation Management

 

The unit of time measure is defaulted to hours, even though the users Absence setup is may be set to another unit (for eg. Days).

This is a known issue and there is no work around.

 

If a journal has been rejected by approver and then deleted by submitter, when the submitter creates a new vacation request the snap does not revert back to the previous journal.

          This is a known issue and there is no work around.

 

At times an error appears when Outlook is closed.

          This is a known issue and there is no work around.

 

If users absence setup is set to No Period in AX, user can enter only one vacation request.

To enter another he has to log into AX and enter one from there and then only can enter another request through the snap

 

Microsoft Dynamics Snap for Timesheet Management

 

Using recurring appointments to create a timesheet does not work.

This is a known issue. A user can however open a single instance of the recurring appointment and use it to create a timesheet.

 

The duration of an all-day appointment is picked up to be 24 hours by default, and not as the work hours set in Outlook.

          This is a known issue and there is no work around.

 

The status of posting is incorrectly displayed on very slow connections.

          The user can view the Timesheet History form to get the exact status of posting.

 

When the AX server is down or not reachable user is not able to use Timesheet.

          This is a known issue and user will not be intimated of the status of the AX server.

 

Timesheet are posted to journals of type WithApproval.

          This is a known issue and there is no work around.

 

Microsoft Dynamics Snap for Customer Journal

 

Results take a very long time to load if the search for account names results in a large number of records

The time to show the results increases linearly with the number of search results returned. Users should narrow down the search by providing additional detail in the search string

 

The Customer Journal Snap-in InfoPath form does not open on a machine where Customer Journal Snap-in is not installed.

          No workaround exists for this issue.

 

Adding a Sales Quotation line item currently is very tedious as the item name is required to be entered at the time of Update and not at the time of creation of the line item.

          This issue has been retained for fixing in future releases.

 

Error messages displayed sometimes are not correct and are not descriptive enough.

          This issue has been retained for fixing in future releases.

 

 

Special characters like &,\,/ etc entered in various fields on the Customer Journal InfoPath form sometimes leads to undefined behavior.

          This issue has been retained for fixing in future releases.

 

When the Customer Journal InfoPath form is closed, it sometimes throws a Runtime error.

Just close the error message box. This issue has been retained for fixing in future releases.
