Overview:  Trafficviewer is a consolidated list of nearly all available traffic camera images in Washington State.  The camera images are stored in different lists, by State Route, County, and by city.  Since Washington State DOT has the most cameras, they are displayed by default, when the program loads. 

  Washington State is divided into 4 quadrants, NW, NE, SW, and SE.  These are selectable by clicking on the small Washington map thumbnail image at the top of the main form.  

  The State Route and City labels are displayed in a box on the top right corner.  These can be clicked on to select a specific route.  As a SR Label is selected, the camera list located below the SR Labels box displays all available camera images for the selected route, or the selected city or county. 

    Nearly all of the files that this program uses are embedded in the executable.  Seperate Files that are not embedded, and registry keys that are created and used by this program are as follows:

  One registry sub key is created whenever an update version is downloaded.  The subkey stores the current location of TrafficViewer.exe, so that the new executable can easily be installed to this location.  This subkey is deleted the first time that the new executable runs, or if the update is cancelled while downloading.  Temporary registry sub key location is: 

    Registry sub key: HKeyLocalMachine\Software\TrafficViewer\

          StringValue Stored in subkey:  (Directory path of current TrafficViewer program)

  An application folder is also created in:  C:\Documents and Settings\UserName\Application Data\TrafficViewer\

Permanent Files used:    1 Settings.dat file, to store the selectable color of the State route labels.

Temp Files used:  The Updater, and new Exe files are stored in a temp folder when downloading, and the temp folder, and all its contents are deleted when the new executable runs.
