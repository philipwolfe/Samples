Standalone JSON Serialization

This sample demonstrates how the DataContractJsonSerializer may be used to serialize and deserialize data in the JSON (JavaScript Object Notation) format. This serialization engine converts JSON data into instances of .NET types and vice versa. The types that are supported are the same as the ones supported by the DataContractSerializer – for more information, see Types Supported by the DataContractSerializer.

The JSON data format is especially useful when writing AJAX-style web applications. Please note that AJAX support in the Windows Communication Foundation is optimized for use with ASP.NET AJAX through the ScriptManager control. For examples of the Windows Communication Foundation used together with ASP.NET AJAX, please see the ASP.NET AJAX Web Service samples.

Note:

The setup procedure and build instructions for this sample are located at the end of this topic.

 

The sample uses a “Person” data contract to demonstrate serialization and deserialization:

 [DataContract]

    internal class Person

    {

        [DataMember]

        internal string name;

 

        [DataMember]

        internal int age;

  }

 

To serialize an instance of this type to JSON, we create the DataContractJsonSerializer and use the WriteObject method to write JSON to a stream:

Person p = new Person();

//Set up Person object...

MemoryStream stream1 = new MemoryStream();

DataContractJsonSerializer ser = new DataContractJsonSerializer(typeof(Person));

ser.WriteObject(stream1, p);

 

When we later examine the contents of the memory stream, it contains valid JSON data:

{“age”:42,”name”:”John”}

 

We then demonstrate deserializing from JSON data into an object. We rewind the stream and then call ReadObject:

Person p2 = (Person)ser.ReadObject(stream1);

 

Examining the p2 object reveals that the JSON data has been deserialized correctly.

 

To set up, build, and run the sample

1. Make sure you have run the one-time setup steps in the main Readme under the root Sample directory

2. Build the solution

3. Run the resulting console application.
