Simple AJAX Service 

This sample demonstrates how the Windows Communication Foundation may be used to create a simple AJAX service (a service that may be consumed using simple JavaScript code from a web browser client). The service uses the HttpTransferContractAttribute attribute to ensure that the service responds to HTTP GET requests, and is configured to use the JSON data format for responses.

Please note that AJAX support in the Windows Communication Foundation is optimized for use with ASP.NET AJAX through the ScriptManager control. When used in this fashion, there is no need to write custom JavaScript code to call the service as in this sample’s ClientPage.htm file – calling a service takes just one line of JavaScript code. For an example of the Windows Communication Foundation used together with ASP.NET AJAX, please see the ASP.NET AJAX Web Service samples.

Note:

The setup procedure and build instructions for this sample are located at the end of this topic.

 

The service is a regular WCF service except with the HttpTransferContractAttribute applied to ensure that the service responds to the HTTP GET verb. Here, GET is used for simplicity only (it is easy to construct a GET URL in JavaScript), but you may also use it for caching.

[ServiceContract(Namespace = "http://Microsoft.Ajax.Samples")]

    public interface ICalculator

    {

        [OperationContract]

        [HttpTransferContract(Method="GET")]

        double Add(double n1, double n2);

        //Other operations omitted…

    }

 

We create an AJAX endpoint on the service by using the webHttpBinding standard binding and the enableWebScript behavior in the service configuration:

    <system.serviceModel>

        <behaviors>

            <endpointBehaviors>

                <behavior name="AjaxBehavior">

                    <enableWebScript />

                </behavior>

            </endpointBehaviors>

        </behaviors>

        <services>

            <service name="Microsoft.Ajax.Samples.CalculatorService">

                <endpoint address="ajaxEndpoint"

                    behaviorConfiguration="AjaxBehavior" binding="webHttpBinding"

                    bindingConfiguration="AjaxBinding" contract="Microsoft.Ajax.Samples.ICalculator" />

            </service>

        </services>

    </system.serviceModel>

 

We then set the data format of the service to JSON instead of XML by setting the binding’s messageEncoding property to Json. This enables us to use the JavaScript “eval” function to parse the response from the service, instead of parsing XML using custom code.

    <system.serviceModel>

        <bindings>

            <webHttpBinding>

                <binding name="AjaxBinding" messageEncoding="Json" />

            </webHttpBinding>

        </bindings>

    </system.serviceModel>

 

This completes the service implementation. The service may now be invoked – for example, by navigating to http://localhost/NetFx35Samples/service.svc/ajaxEndpoint/Add?n1=100&n2=200 after the setup and build steps at the bottom of this readme have been completed. The client webpage, ClientPage.htm, contains JavaScript code to invoke the service whenever the user moves the mouse over one of the operation words on the page. The code to invoke the service looks as follows:

var url = "service.svc/ajaxEndpoint/";

 url = url + operation + "?";

 url = url + "n1=" + document.sampleForm.num1.value + "&";

 url = url + "n2=" + document.sampleForm.num2.value;

 xmlHttp.open("GET",url,true);

 xmlHttp.send('');

 

When the service responds, the response format is JSON and so we can simply use the “eval” function to get a value we can use – in this case, display in a text box:

xmlHttp.onreadystatechange=function()

 {

  if(xmlHttp.readyState==4)

  {

   document.sampleForm.result.value=(eval(xmlHttp.responseText));

  }

 }

 

To set up, build, and run the sample

1. Make sure you have performed the one-time setup steps in the Readme under the root Sample directory, including

a. Ensure you have IIS and ASP.NET installed

b. Set up a NetFx35Samples directory under c:\inetpub\wwwroot and register it as a “NetFx35Samples” vroot and application in IIS setup

2. Build the solution

3. Navigate to http://localhost/NetFx35Samples/ClientPage.htm (do not simply open ClientPage.htm in the browser from the project directory)

4. Ensure you refresh the page in the browser – otherwise, ClientPage.htm from another AJAX sample may be used resulting in errors. The brute-force approach to do this is to run iisreset and re-open the browser
