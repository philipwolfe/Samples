OrderProcessingPolicy

In Orcas, Windows Workflow Foundation (WF) Rules Engine introduced the following functionality:

1.     Added support for operator overloading

2.     Added support for the “new” operator to allow users to new up objects and arrays from WF Rules

3.     Added support for extension methods to make user’s experience calling extension methods from WF Rules compatible with how they code in C#

The OrderProcessingPolicy sample demonstrates simple uses of the new Rules functionalities in Orcas. The sample demonstrates an OrderProcessingPolicy where a customer’s order consisting of an item number of a list of available items, and a zip code are entered. The order will process successfully if both entries are correct. Otherwise, the policy will new up error objects, utilize an overloaded + operator and a predefined extension method to inform the user of the errors.

 Note:

For more information about extension methods, please visit http://msdn2.microsoft.com/en-us/library/ms364047(vs.80).aspx#cs3spec_topic3

 

The sample consists of 3 projects;

OrderErrorLibrary

The OrderErrorLibrary is a class library where an OrderError and OrderErrorCollection classes. An OrderError instance will get created when an invalid input is entered. The library also holds an extention method on OrderErrorCollection that outputs the ErrorText property on all OrderError objects in the OrderErrorCollection

PolicyFromFile

The PolicyFromFile is a custom activity that retrieves/saves rulesets from/to and external .rules file. This activity is used from the OrderProcessingWorkflow to read ruleset from Workflow.rules. This is reusing the PolicyFromFile custom activity located on WF community web site here: http://wf.netfx3.com/files/folders/rules_samples/default.aspx

OrderProcessingPolicy

The OrderProcesssingPolicy is a workflow console application that consists of a single PolicyFromFile activity that consists of 3 rules:

-       invalidItemNum:

Upon entering an invalid item number, in this case if the item number not between 1 and 6, the invalidItemNum rule will

                        1. "new" an OrderError type, and set ErrorText and CustomerName properties on the object,

2. "new" an invalidItemNumrErrorColllection,

3. adds the newly created OrderError instance to the newly created invalidItemNumErrorCollection

-       invalidZip:

Upon entering an invalid zip number, in this case if the zip number length’s < 5 and not between 00600 and 99998, the invalidItemNum rule will

                        1. "new" an OrderError type, and set ErrorText and CustomerName properties on the object,

2. "new" an invalidZipCodeErrorColllection,

3. adds the newly created OrderError instance to the newly created invalidZipCodeErrorCollection

-       displayErrors:

If either invalidItemNumErrorCollection or invalidZipCodeErrorCollection is not null, the rule will

1.     Call the overloaded + operator to copy the contents of invalidItemNumErrorCollection and invalidZipCodeErrorCollection to a invalidOrdersCollection OrderErrorCollection instance

2.     Call the PrintOrderErrors extension method on invalidOrdersCollection and outputs the ErrorText property on all orderError objects in invalidOrdersCollection

To set up, build, and run the sample

The sample will only run on Orcas CTP since its demonstrating new functionalities introduced in Orcas and not available in earlier versions.

1.     Open OrderProcessingPolicy.sln in Visual Studio Codename “Orcas”

2.     Build all projects

3.     Associate the PolicyFromFileActivity in the OrderProcessingPolicy project to the .rules file. You can do this by double clicking on the activity PolicyFromFileActivity and click OK on the Rule Set Editor dialog box. This will set the RulesFileName property on the PolicyFromFileActivity to the full path of the Workflow.rules file located in the OrderProcessingPolicy project folder

4.     Build OrderProcessingPolicy project

5.     Click Run
