RSS and Atom Syndication Sample 

This sample demonstrates how to use WCF to expose server side data as RSS 2.0 or ATOM 1.0 syndication feeds. This data is made available to clients over HTTP GET.

Note:

The setup procedure and build instructions for this sample are located at the end of this topic.

 

Details

WCF models syndication feeds as service operations that return a special data type, System.ServiceModel.Syndication.SyndicationFeed. Instances of SyndicationFeed can be serialized into both the RSS 2.0 and Atom 1.0 formats. The contract used in this sample is illustrated below:

    [ServiceContract(Namespace = "")]

    interface IDiagnosticsService

    {

        [OperationContract]

        //The [HttpTransferContract] attribute controls how WCF dispatches

        //HTTP requests to service operations based on URI suffix

        //(the part of the request URI after the endpoint address)

        //and HTTP method (GET, POST, PUT, etc).

        [HttpTransferContract( Path="", Method="GET" )]

        SyndicationFeed GetProcesses();

    }

 

Notice that the GetProcesses() operation is annotated with an instance of the HttpTransferContractAttribute, which allows you to control how WCF dispatches raw HTTP messages to service operations. HttpTransferContractAttribute allows you to associate each operation with a URI suffix and an HTTP method. The combination of suffix and method will be used to match incoming HTTP requests to service operations.

Hosting the service

Like any WCF service, syndication feeds can be self-hosted or hosting in IIS. Each endpoint hosting a syndication feed must carry an instance of SyndicationBehavior. The following code shows how to use the self-host API’s to create a ServiceHost and add a syndication endpoint to it:

      ServiceHost host = new ServiceHost(typeof(ProcessService), new Uri("http://localhost:8080/diagnostics"));

 

      //Add an endpoint that serves Atom 1.0 content

      ServiceEndpoint atomEndpoint = host.AddServiceEndpoint(typeof(IDiagnosticsService), new WebHttpBinding(), "atom");

      atomEndpoint.Behaviors.Add(new SyndicationBehavior( SyndicationVersion.Atom10 ));

 

      //Add an endpoint that serves Rss 2.0 content

      ServiceEndpoint rssEndpoint = host.AddServiceEndpoint(typeof(IDiagnosticsService), new WebHttpBinding(), "rss");

      rssEndpoint.Behaviors.Add(new SyndicationBehavior(SyndicationVersion.Rss20));

 

      host.Open();

 

Consuming the service

Because this service receives requests using standard HTTP GET, you can use any RSS or ATOM-aware client to consume the service. For example, you can view the output of this service by navigating to http://localhost:8080/diagnostics/GetProcesses in an RSS-aware browser such as Internet Explorer 7.
