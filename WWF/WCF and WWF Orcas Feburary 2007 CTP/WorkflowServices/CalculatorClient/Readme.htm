Calculator Client

The sample is the client that is used with the the state machine workflow service or the Durable calculator service.

 

Note:

The setup procedure and build instructions for this sample are located at the end of this topic.

 

Details

The calculator client application is a WPF calculator UI that can be used to send operations to the calculator state machine service. The Calculator service can be used with different protocols, WsHTTPContextBinding or plain wsHTTPBinding with cookies enabled. You can switch between the bindings and still communicate with the same workflow service instance.

You can also try to power on do some operations and close the calculator without power off. You can then power on and you will find that the calculator will start talking to the workflow instance that it was talking to before. The context is stored in a file called context.ctx which is stored in the bin directory of the Calculator Client application. In order to start new you can delete the context.ctx file.

 

Code

The ICalculator Service contract

 

[ServiceContract(Namespace = "http://Microsoft.WorkflowServices.Samples")]

    public interface ICalculator

    {

        [OperationContract()]

        int PowerOn();

        [OperationContract()]

        int Add(int value);

        [OperationContract()]

        int Subtract(int value);

        [OperationContract()]

        int Multiply(int value);

        [OperationContract()]

        int Divide(int value);

        [OperationContract()]

        void PowerOff();

    } 

 

 

 

To set up

Ensure that the state machine workflow service or the durable calculator service is hosted in IIS as per instructions in the respective samples. Whichever one is installed will have its service.svc and web.config files in the %systemdrive%\intetpub\wwwroot\netfx35samples directory. The “service.dll” file will be in %systemdrive%\intetpub\wwwroot\netfx35samples\bin directory. Make sure that the workflow service reference is properly created. If it is undefined when you open the solution then re-create the reference. You can also look at the WSDL of the state machine workflow service or the durable service by pointing your browser at the appropriate address which is http://localhost/netfx35samples/service.svc to look at the wsdl type in  http://localhost/netfx35samples/service.svc?wsdl

 

Please note that currently the ‘=’ simply adds 0 to the current state of the calculator. This may appear as if the ‘=’ sign is not working. In order to see results simply press an operation other then ‘=’ after you type in the value. The appropriate operation will be done with the value you have typed and the current state of the calculator.

 

The calculator client uses a text file called ‘client.ctx to store the context in a durable place the first time a call is made. The file is stored in the \bin directory of your sample. When you re-open the client it looks if the file is present. If it is present it will use the context stored and apply it to the channel that is being created. In case the workflow service has finished and you open the client with the ‘client.ctx’ file still in your \bin directory it will try to apply the context to the channel. You will get an error in this case since the workflow instance to which you want to communicate with is not there. Simply delete the file and try again.

 

There is a know issue in this sample, When you invoke an operation for the first time after recycling IIS you will get a ‘MessageSecurityException’. Simply invoke another operation and it will succeed. The exception is caused by an expired security token on the channel.
