One-Time Setup Instructions for the WCF and WF ORCAS Feb CTP release

The steps below are most common steps to samples and need to be performed once only. All the scripts mentioned below and in the samples are in the Setup directory.

 

1.    Ensure that ASP.NET is set up. For information on how to set up ASP.NET, see Internet Information Service (IIS) Hosting Instructions.

If you installed WCF after installing IIS you must register the appropriate scriptmaps with IIS by running the following command.

Copy Code

"%WINDIR%\Microsoft.Net\Framework\v3.0\Windows Communication Foundation\ServiceModelReg.exe" -r

1.    If you are running Windows XP SP2, Windows Vista, or Windows Server codename "Longhorn" follow the Firewall Instructions for enabling the ports used by the sample.

2.    Open a command window using an account with Administrator privileges. Run the Setupvroot.bat batch file.

o   A virtual directory is created named NetFx35Samples.

o   A disk directory is created named %SystemDrive%\Inetpub\wwwroot\NetFx35Samples and %SystemDrive%\Inetpub\wwwroot\NetFx35Samples\bin.

o   If you prefer to set up these directories manually, see the Virtual Directory Setup Instructions.

Important   This procedure must be performed only once on a machine.

3.    You must set the ACLs on %systemdrive%\inetpub\wwwroot to grant modify permissions to the account under which you are running. Otherwise some post build events can fail when you build Web-hosted samples. Alternatively, you can leave the ACLs as they are and run Visual Studio as an Administrator.

4.    Samples that are self-hosted (not hosted in IIS) require permission to register HTTP addresses on the machine for listening. The permission for an HTTP namespace reservation comes from the user account used to run the sample. By default, administrator accounts have the permission to register any HTTP address. Non-administrator accounts must be granted permission for the HTTP namespaces used by the samples. For information about configuring namespace reservations, see Configuring HTTP and HTTPS.

5.    Run %windir%\microsoft.net\framework\v2.0.50727\aspnet_regiis -i –enable

6.    Run %windir%\microsoft.net\framework\v3.5.60129\Cfx_Installer –i

7.    For the WorkflowServices samples that require persistence

o   Make sure you have SQLExpress installed

o   Add ASPNET as a user of the SQLServer*SQLExpress account

o   Run CreateStores.cmd before running the sampless and DeleteStores.cmd when done running all samples

Note   For security purposes, remove the virtual directory definition and permissions granted in the setup steps when you are finished with the samples by running the batch file named Cleanupvroot.bat.
